Flashcards

Pourquoi `learning_rate=2e-5` (0.00002) est-il recommandé pour le fine-tuning plutôt que la valeur par défaut 1e-3 ?

Voir la réponse
Réponse

Les poids pré-entraînés encodent des représentations utiles. Un learning rate élevé les écrase (catastrophic forgetting). 2e-5 les ajuste délicatement pour la nouvelle tâche.

Dans l'étape de tokenisation, à quoi sert `padding="max_length"` ?

Voir la réponse
Réponse

Les phrases d'un batch ont des longueurs différentes. Le padding ajoute des tokens spéciaux [PAD] pour que toutes atteignent la même longueur — nécessaire pour former des tenseurs rectangulaires.

Flashcards

Définissez l'overfitting. Comment le reconnaître sur des courbes d'entraînement ?

Voir la réponse
Réponse

Le modèle mémorise le dataset d'entraînement au lieu d'apprendre des patterns généralisables. Sur les courbes : la loss d'entraînement continue de baisser, mais la loss de validation commence à remonter.

Citez 4 techniques pour réduire l'overfitting sur un petit dataset d'images.

Voir la réponse
Réponse

1. Data augmentation (flip, rotation, color jitter), 2. Dropout dans la tête de classification, 3. Réduire le learning rate, 4. Early stopping (arrêter quand la val loss remonte), 5. Feature extraction plutôt que fine-tuning complet, 6. Weight decay (régularisation L2 — pénalise les poids trop grands du réseau pour éviter que le modèle mémorise les données d'entraînement).

Qu'est-ce que l'early stopping ?

Voir la réponse
Réponse

Surveiller la val loss pendant l'entraînement et arrêter automatiquement quand elle ne s'améliore plus depuis N epochs consécutives. On restaure les poids du meilleur checkpoint.

Quelle est la différence entre val loss et train loss, et pourquoi surveille-t-on les deux ?

Voir la réponse
Réponse

Train loss : erreur sur les données d'entraînement (que le modèle a vues). Val loss : erreur sur des données jamais vues. Si train loss baisse mais val loss monte → overfitting. On surveille les deux pour détecter ce décrochage.
