Exercice 2 — Interpréter des coefficients Ridge

Un modèle Ridge (alpha = 10) a été entraîné sur des données normalisées pour prédire le salaire annuel d'un employé. Voici les coefficients obtenus :

Variable Coefficient Ridge (alpha=10) Coefficient sans régularisation
années_expérience +12 400 +18 900
niveau_diplôme +8 200 +9 100
taille_entreprise +1 800 +14 300
secteur_tech +6 500 +7 200
distance_bureau −320 −14 800
nb_langues +280 +16 400

Questions :

2.1 — Quelle variable a subi la plus forte réduction relative de son coefficient ? Que peut-on en déduire sur sa relation avec les autres variables ?

Votre réponse :

2.2 — Les variables distance_bureau et nb_langues ont des coefficients très faibles en Ridge. Que ferait Lasso avec ces variables ?

Votre réponse :

2.3 — années_expérience reste le coefficient le plus fort. Que signifie ce coefficient de +12 400 dans le contexte de données normalisées ?

Votre réponse :

2.4 — Si on augmente alpha de 10 à 100, comment les coefficients vont-ils évoluer ? Nommez deux risques opposés liés à un alpha trop grand.

Votre réponse :

Corrigé

2.1 — distance_bureau : de −14 800 à −320 (réduction de 97.8%) et nb_langues : de +16 400 à +280 (réduction de 98.3%). Ces deux variables sont probablement très corrélées avec d'autres (multicolinéarité) — la régression ordinaire leur avait attribué des coefficients instables et extrêmes pour compenser des relations croisées. Ridge les ramène à une contribution plus raisonnable.

2.2 — Lasso les mettrait probablement exactement à 0 — elles seraient éliminées du modèle, ce qui le rendrait plus parcimonieux (uniquement les 4 variables restantes).

2.3 — Sur des données normalisées, le coefficient représente l'effet d'un écart-type d'expérience sur le salaire. Ce n'est pas directement "+12 400 € par année" — c'est "+12 400 € pour un écart-type d'années d'expérience". Il faut multiplier par l'écart-type réel des années d'expérience pour revenir à l'interprétation brute.

2.4 — Tous les coefficients vont diminuer vers 0. Deux risques : (1) underfitting si alpha est trop grand — le modèle ne peut plus capturer la vraie structure des données ; (2) perte d'interprétabilité — tous les coefficients deviennent si petits qu'on ne peut plus dire quelle variable compte vraiment.

Exercice 3 — Détecter overfitting et underfitting

Pour chaque description de résultats, indiquez : Underfitting, Overfitting, ou Bon ajustement. Proposez une action corrective.

Situation A

Un modèle de régression polynomiale degré 8 est entraîné pour prédire le chiffre d'affaires mensuel. - Score R² sur entraînement : 0.98 - Score R² sur validation : 0.51 - Les courbes d'apprentissage montrent un grand écart stable entre train et validation.

Un modèle Ridge prédit la demande en énergie. - Score R² entraînement : 0.71 - Score R² validation : 0.69 - Les deux courbes d'apprentissage sont proches mais toutes les deux autour de 0.70.

Diagnostic : Action corrective :

Situation C

Un modèle Lasso (alpha = 0.001) prédit le prix d'une voiture d'occasion. - Score R² entraînement : 0.94 - Score R² validation : 0.91 - En augmentant la taille des données d'entraînement, les deux scores restent stables.

Diagnostic : Action corrective :

Situation D

Un modèle Ridge (alpha = 1 000) est appliqué sur un problème avec 3 variables. - Score R² entraînement : 0.38 - Score R² validation : 0.36 - Les deux scores sont proches mais très bas.

Diagnostic : Action corrective :

Corrigé

Situation A → Overfitting. Grand écart train/validation stables = le modèle mémorise. Action : réduire le degré du polynôme (essayer degré 2 ou 3) et augmenter la régularisation (Ridge avec alpha plus grand).

Situation B → Underfitting léger. Les deux courbes sont proches (pas d'overfitting) mais le score est trop bas pour être utile. Action : augmenter la complexité du modèle (ajouter des features, essayer un polynôme degré 2, ou diminuer alpha pour relâcher la contrainte).

Situation C → Bon ajustement. R² élevé, écart train/validation faible (3 points), stable avec plus de données. Action : rien — le modèle est prêt à être utilisé. Vérifier les métriques métier (RMSE en €).

Situation D → Underfitting par régularisation excessive. Alpha = 1 000 est trop fort pour seulement 3 variables — tous les coefficients sont tassés si près de 0 que le modèle ne peut rien apprendre. Action : diminuer drastiquement alpha (essayer 0.1, 1, 10) et utiliser LassoCV / RidgeCV pour trouver la valeur optimale.

4.1 — Donnez la définition d'un résidu en une phrase.

4.2 — Quelle est la seule différence entre Ridge et Lasso dans leur formule de pénalité ?

4.3 — Complétez : "Lasso peut forcer un coefficient à ________ ; Ridge ne le peut pas."

4.4 — Un modèle performe à 0.99 sur le train et 0.60 en validation. Le problème est-il lié au biais ou à la variance ? Pourquoi ?

4.5 — Expliquez en une phrase pourquoi il faut normaliser les données avant d'appliquer Ridge ou Lasso.

4.6 — PolynomialFeatures(degree=2) appliqué à une variable x produit quelles nouvelles colonnes ?

4.7 — Sur une courbe d'apprentissage, si les courbes train et validation sont toutes les deux basses et très proches, de quoi s'agit-il ?

4.8 — Quelle métrique d'évaluation est la plus robuste aux valeurs aberrantes (outliers) : RMSE ou MAE ? Pourquoi ?

Corrigé Checkpoint

4.1 — Un résidu est la différence entre la valeur réelle observée et la valeur prédite par le modèle (résidu = yréel − yprédit).

4.2 — Ridge pénalise le carré des coefficients (L2) ; Lasso pénalise leur valeur absolue (L1).

4.3 — Lasso peut forcer un coefficient à zéro ; Ridge ne le peut pas.

4.4 — Variance forte (overfitting). Le modèle a mémorisé les données d'entraînement — il généralise très mal. Le biais fort se manifeste par de mauvais scores sur les deux ensembles.

4.5 — Parce que Ridge et Lasso pénalisent numériquement les grands coefficients : sans normalisation, une variable avec des valeurs en milliers aurait un coefficient naturellement petit et subirait moins de pénalité qu'une variable en unités, ce qui fausserait la régularisation.

4.6 — x et x² (plus la constante si include_bias=True).

4.7 — Underfitting — le modèle est trop simple pour capturer la relation dans les données.

4.8 — MAE, parce qu'elle prend la valeur absolue de l'erreur sans la mettre au carré. RMSE met au carré les grandes erreurs, ce qui les amplifie davantage.
