21 — Régression linéaire : ce qu'elle fait vraiment

La régression linéaire répond à une question précise : parmi toutes les droites (ou hyperplans) qu'on peut tracer à travers un nuage de points, laquelle colle le mieux aux données ?

Hyperplan : en 2D c'est une droite, en 3D c'est un plan, au-delà c'est un hyperplan — tous ces mots désignent la même idée : la surface plate la plus simple possible dans l'espace des données.

Résidu : la différence entre ce que le modèle prédit et la valeur réelle. Si un appartement vaut 350 000 € et que le modèle prédit 320 000 €, le résidu est −30 000 €. Les résidus peuvent être positifs ou négatifs.

L'objectif de la régression : trouver les coefficients qui minimisent la somme des résidus au carré (on met au carré pour que les erreurs positives et négatives ne se compensent pas, et pour pénaliser davantage les grosses erreurs).

Situation réelle — Nexity

Le groupe immobilier Nexity entraîne un modèle pour prédire le prix de vente d'un appartement à partir de trois variables : la surface en m², l'étage, et la distance à la station de métro la plus proche.

import pandas as pd
from sklearn.linear_model import LinearRegression

# Données fictives mais réalistes
df = pd.DataFrame({
    "surface_m2":      [32, 48, 55, 70, 85, 90, 110, 130],
    "etage":           [1,  3,  2,  5,  7,  4,  8,   10 ],
    "dist_metro_m":    [800,400,200,600,100,900,300,  150],
    "prix_euros":      [220000, 310000, 340000, 420000,
                        510000, 390000, 620000, 750000]
})

X = df[["surface_m2", "etage", "dist_metro_m"]]
y = df["prix_euros"]

model = LinearRegression()
model.fit(X, y)

# Lire les coefficients
for feature, coef in zip(X.columns, model.coef_):
    print(f"{feature:20s} → {coef:+.1f} €")
print(f"{'Constante':20s} → {model.intercept_:+.1f} €")
# surface_m2           → +5 200.0 €  (chaque m² supplémentaire ajoute ~5 200 €)
# etage                → +8 500.0 €  (chaque étage ajoute ~8 500 €)
# dist_metro_m         →   -120.0 €  (chaque mètre de distance enlève ~120 €)

Lire un coefficient : le coefficient de surface_m2 (+5 200 €) signifie "toutes autres choses égales, un m² supplémentaire est associé à un prix plus élevé de 5 200 €". Ce n'est pas une causalité, c'est une corrélation apprise sur les données.

Flashcards Section 1

Q : Qu'est-ce qu'un résidu ? R : La différence entre la valeur réelle et la valeur prédite par le modèle.

Q : Pourquoi minimise-t-on les résidus au carré et pas leur somme simple ? R : Parce que les résidus positifs et négatifs se compenseraient. Mettre au carré force tous les termes à être positifs, et pénalise davantage les grosses erreurs.

Q : Que signifie un coefficient négatif dans une régression linéaire ? R : Que lorsque cette variable augmente, la prédiction diminue (toutes autres variables égales par ailleurs).

Q : À quoi sert l'ordonnée à l'origine (constante) dans une régression ? R : C'est la prédiction quand toutes les variables valent zéro — un point de départ théorique.

32 — Le problème de l'overfitting en régression

Un modèle apprend sur un jeu de données d'entraînement. Le vrai objectif est qu'il généralise — qu'il soit utile sur des données qu'il n'a jamais vues. Deux types d'erreurs peuvent empêcher cela.

Biais : erreur systématique due à un modèle trop simple. Le modèle n'a pas assez de flexibilité pour capturer la vraie relation dans les données. Il fait des erreurs même sur les données d'entraînement. On parle d'underfitting (sous-ajustement).

Variance : erreur due à un modèle trop sensible aux détails du jeu d'entraînement. Le modèle a appris les données par cœur — y compris le bruit et les coïncidences. Il performe très bien sur l'entraînement mais échoue sur de nouvelles données. On parle d'overfitting (sur-ajustement).

Underfitting (biais fort) Droite — ne suit pas la courbe

Bon ajustement Parabole — suit la tendance

Overfitting (variance forte) Courbe degree-7 — mémorise le bruit

Situation réelle — AXA

Une équipe data chez AXA construit un modèle de prédiction du coût des sinistres automobiles. Le modèle atteint 99 % de précision sur les données d'entraînement. En production, il tombe à 61 %. Diagnostic : le modèle a mémorisé les particularités des données historiques (accidents survenus une année précise, effets saisonniers locaux, anomalies de saisie) au lieu d'apprendre la relation générale entre le profil du conducteur et le risque. C'est de l'overfitting.

La solution ne consiste pas à ajouter plus de données au hasard — elle consiste à contraindre le modèle pour qu'il ne puisse pas mémoriser. C'est exactement ce que font Ridge et Lasso.

Flashcards Section 2

Q : Quelle est la différence entre biais et variance ? R : Le biais est une erreur systématique (modèle trop simple, rate la tendance). La variance est une sensibilité excessive aux données d'entraînement (modèle trop complexe, mémorise le bruit).

Q : Un modèle à 99% sur le train et 61% en prod, c'est quoi ? R : De l'overfitting — variance trop forte.

Q : Comment reconnaître l'underfitting sur une courbe ? R : Le modèle fait des erreurs importantes même sur les données d'entraînement.

Q : Pourquoi augmenter la complexité du modèle n'est pas toujours la bonne réponse ? R : Parce qu'au-delà d'un certain point, la complexité supplémentaire sert à mémoriser le bruit plutôt qu'à apprendre la vraie structure.

43 — Ridge (L2) : pénaliser les grands coefficients

Définition

La régression ordinaire cherche les coefficients qui minimisent les erreurs sur les données d'entraînement. Ridge fait la même chose, mais ajoute une pénalité : chaque coefficient qui prend une grande valeur "coûte" quelque chose lors de l'entraînement.

Cette pénalité est proportionnelle au carré de chaque coefficient — d'où le nom L2 (norme L2 = racine de la somme des carrés). Le modèle arrive donc à un compromis : réduire les erreurs sur les données, mais en évitant les coefficients très grands.

Paramètre alpha : c'est vous qui décidez du poids de cette pénalité. - alpha = 0 : pas de pénalité — identique à la régression ordinaire - alpha = 0.1 : légère pénalité — les coefficients sont légèrement réduits - alpha = 100 : pénalité forte — tous les coefficients sont tassés près de zéro

Ce que Ridge ne fait pas : il ne met jamais un coefficient exactement à zéro. Toutes les variables restent dans le modèle, mais avec des coefficients plus petits.

Quand choisir Ridge

  • Toutes les variables semblent utiles (on ne veut pas en éliminer)
  • Les variables sont corrélées entre elles — par exemple, la surface et le nombre de pièces se ressemblent. Sans Ridge, la régression ordinaire peut donner un coefficient très grand à l'une et très négatif à l'autre pour se "compenser", ce qui est instable. Ridge calme ce comportement.

Ce phénomène s'appelle la multicolinéarité : deux variables qui mesurent presque la même chose. Ridge la gère élégamment.

from sklearn.linear_model import Ridge
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline

# IMPORTANT : Ridge est sensible à l'échelle des variables.
# On normalise d'abord (StandardScaler) pour que chaque variable
# parte sur un pied d'égalité.

pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("ridge", Ridge(alpha=10.0))
])

pipeline.fit(X, y)

ridge_model = pipeline.named_steps["ridge"]
for feature, coef in zip(X.columns, ridge_model.coef_):
    print(f"{feature:20s} → {coef:+.1f}")

Choisir alpha : on utilise la validation croisée. sklearn propose RidgeCV qui teste plusieurs valeurs d'alpha automatiquement :

from sklearn.linear_model import RidgeCV
import numpy as np

alphas = np.logspace(-3, 3, 50)  # 50 valeurs de 0.001 à 1000
model_cv = RidgeCV(alphas=alphas, cv=5)
model_cv.fit(X_scaled, y)
print(f"Meilleur alpha : {model_cv.alpha_:.4f}")

Flashcards Section 3

Q : Que signifie "pénalité L2" dans Ridge ? R : On ajoute à la fonction de perte le carré de chaque coefficient, multiplié par alpha. Les grands coefficients deviennent coûteux à apprendre.

Q : Ridge peut-il mettre un coefficient exactement à 0 ? R : Non — Ridge réduit les coefficients vers 0 mais ne les annule jamais complètement.

Q : Qu'est-ce que la multicolinéarité et pourquoi Ridge la gère-t-elle bien ? R : Multicolinéarité = deux features très corrélées. La régression ordinaire devient instable. Ridge répartit les coefficients de façon équilibrée entre ces features.

Q : Pourquoi faut-il normaliser les données avant Ridge ? R : Parce que Ridge pénalise les grands coefficients numériquement. Une variable en milliers (prix) aurait un coefficient plus petit qu'une variable en unités (étages) — sans normalisation, la pénalité est injuste.

54 — Lasso (L1) : sélection automatique de features

Définition

Lasso (Least Absolute Shrinkage and Selection Operator) fonctionne comme Ridge mais avec une pénalité différente : au lieu de pénaliser le carré des coefficients (L2), il pénalise leur valeur absolue (L1 = somme des valeurs absolues).

Cette différence, qui semble mineure, a une conséquence radicale : Lasso peut forcer des coefficients exactement à zéro. Il fait donc de la sélection de features automatique — certaines variables sont complètement exclues du modèle.

Un modèle avec beaucoup de coefficients à zéro est dit sparse (creux) — il n'utilise qu'un sous-ensemble des variables disponibles.

Ridge (L2) — tend vers 0 0 0.001 0.01 0.1 1 10 100 Les courbes s'approchent de 0 sans l'atteindre

Lasso (L1) — atteint 0 0 0.001 0.01 0.1 1 10 100 = 0 (éliminées) Les features tombent à exactement 0 — sélection automatique

Quand choisir Lasso

  • Vous avez beaucoup de variables et suspectez que seules quelques-unes comptent vraiment
  • Vous souhaitez un modèle interprétable avec peu de variables retenues
  • Exemple : une étude médicale avec 500 marqueurs biologiques — Lasso en garde peut-être 20

Code sklearn avec lecture des variables éliminées

from sklearn.linear_model import Lasso, LassoCV
from sklearn.preprocessing import StandardScaler
import numpy as np

# Trouver le meilleur alpha par validation croisée
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

lasso_cv = LassoCV(alphas=np.logspace(-4, 2, 100), cv=5)
lasso_cv.fit(X_scaled, y)

print(f"Meilleur alpha : {lasso_cv.alpha_:.4f}")

# Lire les coefficients et identifier les variables éliminées
for feature, coef in zip(X.columns, lasso_cv.coef_):
    status = "ÉLIMINÉE" if coef == 0 else f"{coef:+.1f}"
    print(f"{feature:20s} → {status}")

Flashcards Section 4

Q : Quelle est la différence mathématique entre Ridge et Lasso ? R : Ridge pénalise le carré des coefficients (L2), Lasso pénalise leur valeur absolue (L1).

Q : Pourquoi Lasso peut-il mettre un coefficient à exactement 0 et pas Ridge ? R : La pénalité en valeur absolue crée une discontinuité géométrique (coin du losange) qui force des solutions exactement à 0. Le cercle de Ridge n'a pas ce coin.

Q : Qu'est-ce qu'un modèle "sparse" ? R : Un modèle dont beaucoup de coefficients valent 0 — il n'utilise qu'un sous-ensemble des variables disponibles.

Q : Dans quel cas Lasso est-il préférable à Ridge ? R : Quand on a beaucoup de features dont une grande partie sont probablement inutiles, et qu'on veut un modèle sélectif et interprétable.

65 — Régression polynomiale : modéliser les courbes

Définition

Certaines relations entre variables ne sont pas linéaires. La consommation électrique d'un logement en fonction de la température forme une courbe en U : elle augmente quand il fait très froid (chauffage) et quand il fait très chaud (climatisation). Une droite ne peut pas capturer cela.

La régression polynomiale résout ce problème sans changer d'algorithme : on crée de nouvelles variables à partir des variables existantes (x², x³, x⁴...), puis on fait une régression linéaire sur ces nouvelles variables. On appelle ça du feature engineering — l'art de créer des variables utiles à partir de celles qu'on a.

Feature engineering : transformer ou combiner les variables existantes pour donner plus d'information au modèle. La régression polynomiale en est l'exemple le plus simple.

Ce n'est pas un nouvel algorithme : le modèle final reste une régression linéaire — mais sur des variables transformées.

Degré 1 — Underfitting Droite — rate la courbe en U

Degré 2 — Bon ajustement Parabole — capture le U

Degré 10 — Overfitting Ondule partout — mémorise le bruit Performant sur train, catastrophique en prod

Code sklearn avec PolynomialFeatures expliqué

PolynomialFeatures est un transformateur : il prend une variable x et crée x, x², x³... jusqu'au degré demandé. Si on a deux variables x₁ et x₂, il crée aussi les interactions x₁·x₂ (une interaction = le produit de deux variables, pour capturer l'effet combiné de l'une sur l'autre).

from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import Ridge
from sklearn.pipeline import Pipeline
import numpy as np

# Données EDF fictives : température (°C) et consommation (kWh/jour)
temperatures = np.array([-10, -5, 0, 5, 10, 15, 20, 25, 30, 35, 40])
consommation = np.array([85, 72, 60, 48, 38, 32, 30, 34, 42, 55, 68])

X = temperatures.reshape(-1, 1)
y = consommation

# Pipeline :
# 1. PolynomialFeatures(degree=2) transforme [x] en [1, x, x²]
# 2. Ridge régularise pour éviter l'overfitting
pipeline = Pipeline([
    ("poly",  PolynomialFeatures(degree=2, include_bias=False)),
    ("model", Ridge(alpha=1.0))
])

pipeline.fit(X, y)

# Vérifier les nouvelles features créées
poly = pipeline.named_steps["poly"]
print("Features créées :", poly.get_feature_names_out(["temperature"]))
# ['temperature', 'temperature^2']

# Prédire la consommation à 22°C
pred = pipeline.predict([[22]])
print(f"Consommation prédite à 22°C : {pred[0]:.1f} kWh/jour")

Situation réelle — EDF

EDF modélise la consommation électrique journalière de la France en fonction de la température. La relation est bien en U : sous 15°C le chauffage s'emballe, au-dessus de 25°C la climatisation prend le relais. Une régression linéaire prédirait une consommation monotone (toujours plus ou toujours moins avec la température) — complètement fausse. Un polynôme de degré 2 capture cette réalité avec un R² supérieur à 0.92, contre 0.41 pour la droite.

Flashcards Section 5

Q : Qu'est-ce que le "feature engineering" dans le contexte de la régression polynomiale ? R : On crée de nouvelles variables (x², x³, produits de variables) à partir des variables existantes pour permettre à un modèle linéaire de capturer des relations non-linéaires.

Q : Pourquoi la régression polynomiale reste-t-elle une "régression linéaire" ? R : Parce que le modèle est linéaire en ses coefficients — il fait la somme pondérée de ses variables d'entrée. Les variables d'entrée sont juste x, x², x³ plutôt que x seulement.

Q : Quel est le danger d'utiliser un degré trop élevé ? R : Overfitting sévère — la courbe ondule pour passer par chaque point d'entraînement et devient inutilisable sur de nouvelles données.

Q : Comment limiter l'overfitting en régression polynomiale ? R : Combiner PolynomialFeatures avec Ridge ou Lasso, et choisir le degré par validation croisée.

Courbes d'apprentissage — détecter underfitting et overfitting

Une courbe d'apprentissage montre la performance du modèle en fonction de la taille du jeu d'entraînement, séparément pour les données d'entraînement et de validation. C'est l'outil de diagnostic le plus utile pour savoir si un problème vient du modèle ou des données.

Underfitting Taille du jeu d'entraînement → Score 1.0 0.7 0.4 Train Validation Les deux courbes restent basses → modèle trop simple

Overfitting Taille du jeu d'entraînement → 1.0 0.7 0.4 Gap Train (haute) Validation (basse) Grand écart train / validation → modèle mémorise, pas généralise

Métriques d'évaluation

RMSE Root Mean Squared Error √( Σ(y−ŷ)² / n ) Dans la même unité que y Pénalise les grosses erreurs 0 = parfait ; plus bas = mieux

MAE Mean Absolute Error Σ|y−ŷ| / n Dans la même unité que y Robuste aux outliers 0 = parfait ; plus bas = mieux

R² Coefficient de détermination 1 − SSres / SStot Sans unité, entre −∞ et 1 1 = parfait ; 0 = inutile Proportion de variance expliquée 

from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score
import numpy as np

y_true = [350000, 420000, 510000, 620000]
y_pred = [330000, 445000, 495000, 640000]

rmse = np.sqrt(mean_squared_error(y_true, y_pred))
mae  = mean_absolute_error(y_true, y_pred)
r2   = r2_score(y_true, y_pred)

print(f"RMSE : {rmse:,.0f} €")   # Erreur quadratique moyenne
print(f"MAE  : {mae:,.0f} €")    # Erreur absolue moyenne
print(f"R²   : {r2:.3f}")        # Proportion de variance expliquée

Flashcards Section 6

Q : Sur une courbe d'apprentissage, comment reconnaît-on l'underfitting ? R : Les deux courbes (train et validation) restent basses — le modèle est trop simple pour les deux.

Q : Quelle est la différence entre RMSE et MAE ? R : RMSE met au carré avant de faire la moyenne, ce qui pénalise davantage les grosses erreurs. MAE traite toutes les erreurs proportionnellement à leur taille.

Q : Un R² de 0.85 veut dire quoi ? R : Le modèle explique 85% de la variance de la variable cible. Les 15% restants sont du bruit ou des facteurs non capturés.

Q : Si le score train est 0.97 et le score validation 0.62, que faire en priorité ? R : Réduire la complexité du modèle ou augmenter la régularisation (alpha plus grand dans Ridge ou Lasso).
